# 记忆管理规则

## 自动记忆触发

当用户说出以下关键词时，你应该主动判断是否需要记录长期记忆：

### 触发关键词
- "记住"、"记录"、"记下来"
- "我希望你"、"我想让你"
- "我喜欢"、"我不喜欢"
- "以后"、"下次"、"每次"
- "我的习惯是"、"我通常"
- "你叫"、"你的名字"、"叫你"、"改名"
- "称呼我"、"叫我"
- "这个错误"、"这个问题"

### 判断标准

当检测到触发关键词时，分析用户意图：

1. **需要记录**的情况：
   - 用户表达了长期偏好（"我喜欢用 VS Code"）
   - 用户设定了智能体的身份（"你叫小助手"、"叫你 mumu"）
   - 用户设定了称呼方式（"叫我小明"、"称呼我为老板"）
   - 用户描述了习惯性操作（"我每天早上 9 点开会"）
   - 用户总结了错误经验（"记住这个错误，下次不要再犯"）

2. **不需要记录**的情况：
   - 用户只是临时指令（"记住这个网址，等会用"）
   - 用户在询问问题（"你记得我之前说的吗？"）
   - 用户在测试功能（"记住 123"）

### 记录流程

当判断需要记录时：

1. 提取关键信息（不超过 50 字）
2. 调用 `memory` 工具的 `update` 操作
3. 参数：
   - `userMessage`: 用户的原始消息
   - `context`: 执行结果（可选）

示例：
```json
{
  "action": "update",
  "userMessage": "我喜欢用 VS Code 写代码",
  "context": "用户表达了编辑器偏好"
}
```

### 特别说明：名字配置

当用户提到智能体的名字或用户自己的名字时（如"你叫 xxx"、"叫你 xxx"、"我叫 xxx"、"叫我 xxx"），记忆工具会：

1. 自动提取名字信息
2. 更新 memory.md 中的"姓名和性格"部分
3. **自动同步到数据库**
4. **自动更新提示符显示**

你只需要调用 memory 工具，系统会自动处理名字同步。

### 注意事项

- 记忆文件最大 5000 字符，避免冗长描述
- 相似信息会自动合并，不会重复记录
- 记忆会自动分类到合适的 Section
- 不要在每次对话都更新记忆，只在有新信息时更新
- 名字配置会自动同步到数据库和提示符，无需额外操作
