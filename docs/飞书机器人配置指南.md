# 飞书机器人配置指南

本文档介绍如何配置 DeepBot 的飞书连接器，使其能够通过飞书接收和发送消息。

## 前置条件

1. 拥有飞书企业管理员权限
2. 已安装 DeepBot Terminal

## 配置步骤

### 1. 创建飞书企业自建应用

1. 访问[飞书开放平台](https://open.feishu.cn/)
2. 登录后，点击「创建企业自建应用」
3. 填写应用信息：
   - 应用名称：DeepBot（或自定义名称）
   - 应用描述：AI 助手机器人
   - 应用图标：上传应用图标（可选）

### 2. 获取应用凭证

1. 在应用详情页，进入「凭证与基础信息」
2. 记录以下信息：
   - **App ID**（应用 ID）
   - **App Secret**（应用密钥）

### 3. 配置应用权限

在「权限管理」页面，添加以下权限：

#### 必需权限

- `im:message` - 获取与发送单聊、群组消息
- `im:message:send_as_bot` - 以应用的身份发消息
- `im:chat` - 获取群组信息

#### 可选权限（推荐）

- `contact:user.base:readonly` - 获取用户基本信息
- `im:chat:readonly` - 获取群组信息

#### 文档读取权限（可选）

如果需要让 DeepBot 读取飞书文档和电子表格内容，需要添加以下权限：

**飞书文档权限**：
- `docx:document:readonly` - 读取云文档内容（docx、docs、wiki）
- `drive:drive:readonly` - 访问云空间文件

**电子表格权限**：
- `sheets:spreadsheet:readonly` - 读取电子表格内容（sheets）
- `drive:drive:readonly` - 访问云空间文件（如果还没添加）

**说明**：添加这些权限后，用户在消息中发送飞书文档或电子表格链接时，DeepBot 会自动读取内容并进行分析。

### 4. 配置事件订阅

1. 进入「事件订阅」页面
2. 选择「使用长连接接收事件」（推荐）
   - 无需配置请求地址
   - SDK 会自动建立 WebSocket 连接
3. 订阅以下事件：
   - `im.message.receive_v1` - 接收消息

**注意**：
- 长连接模式仅支持企业自建应用
- 无需配置 Encrypt Key（SDK 自动处理）
- 无需配置请求地址

### 5. 发布应用

1. 在「版本管理与发布」页面
2. 创建版本并提交审核
3. 审核通过后，发布应用到企业

### 6. 在 DeepBot 中配置

1. 启动 DeepBot Terminal
2. 进入「设置」→「连接器配置」
3. 选择「飞书」连接器
4. 填写配置信息：
   - **App ID**：从步骤 2 获取
   - **App Secret**：从步骤 2 获取
   - **机器人名称**：自定义显示名称（如 "DeepBot"）
   - **DM 策略**：选择私聊消息处理策略
     - `pairing`：需要配对码授权（推荐）
     - `allowlist`：仅允许白名单用户
     - `open`：允许所有用户
   - **群组策略**：选择群组消息处理策略
     - `open`：允许所有群组（推荐）
     - `allowlist`：仅允许白名单群组
     - `disabled`：禁用群组消息
   - **需要 @提及**：群组中是否需要 @机器人（推荐开启）

5. 点击「保存」
6. 点击「启动」按钮启动连接器

## 使用说明

### 私聊使用

#### Pairing 模式（推荐）

1. 在飞书中搜索并添加机器人
2. 发送任意消息
3. 机器人会返回配对码（如：`ABC123`）
4. 管理员在 DeepBot Terminal 中批准：
   - 进入「设置」→「连接器配置」→「飞书」
   - 切换到「Pairing 管理」标签页
   - 找到对应的配对记录，点击「批准」按钮
5. 批准后用户即可正常对话

#### Allowlist 模式

1. 管理员在配置中添加用户 ID 到白名单
2. 白名单用户可直接与机器人对话

### 群组使用

1. 将机器人添加到群组
2. 在群组中 @机器人 发送消息
3. 机器人会回复消息

### 文件发送功能

DeepBot 支持通过飞书发送图片和文件：

1. **发送图片**：Agent 可以使用 `connector_send_image` 工具发送图片
   - 支持格式：JPG、PNG、GIF、BMP、WebP
   - 可选添加图片说明文字

2. **发送文件**：Agent 可以使用 `connector_send_file` 工具发送文件
   - 支持任意文件类型
   - 可自定义文件名

**使用示例**：
- "帮我把这个图片发送到飞书"
- "把这个文档发送给用户"
- "发送 /path/to/report.pdf 文件"

**注意**：文件发送功能仅在连接器会话中可用（即从飞书发起的对话）。

### 飞书文档和电子表格读取功能

DeepBot 支持自动读取飞书文档和电子表格内容：

1. **自动检测**：当用户在消息中发送飞书文档或电子表格链接时，DeepBot 会自动检测
2. **读取内容**：自动调用飞书 API 读取文档或表格的内容
3. **附加到消息**：将内容附加到用户消息中，供 AI 分析

**支持的文档类型**：
- 飞书文档（docx）
- 飞书旧版文档（docs）
- 飞书知识库（wiki）
- 飞书电子表格（sheets）- 会读取所有工作表的数据

**使用示例**：
- 用户：发送飞书文档链接 + "帮我总结这个文档"
- 用户：发送飞书电子表格链接 + "分析这个表格的数据"
- DeepBot：自动读取内容并进行分析

**权限要求**：需要在飞书开放平台配置相应权限（见上方「文档读取权限」部分）。

## 技术说明

### 使用飞书官方 SDK

DeepBot 使用飞书官方 Node.js SDK (`@larksuiteoapi/node-sdk`) 实现：

- **长连接模式**：通过 WebSocket 接收事件，无需配置公网地址
- **自动 Token 管理**：SDK 自动获取和刷新 access_token
- **事件分发**：使用 EventDispatcher 处理消息事件
- **API 调用**：使用 Client 发送消息

### 架构优势

1. **简化配置**：无需配置请求地址、Encrypt Key
2. **本地开发友好**：无需内网穿透工具
3. **自动重连**：SDK 自动处理连接断开和重连
4. **类型安全**：完整的 TypeScript 类型定义

## 常见问题

### Q: 长连接模式有什么限制？

A: 
- 仅支持企业自建应用（不支持商店应用）
- 每个应用最多 50 个连接
- 集群模式（不支持广播）

### Q: 如何查看连接状态？

A: 在 DeepBot Terminal 的「连接器配置」页面，可以看到连接器的运行状态。

### Q: 消息发送失败怎么办？

A: 
1. 检查应用权限是否正确配置
2. 检查 App ID 和 App Secret 是否正确
3. 查看 DeepBot 日志获取详细错误信息

### Q: 如何获取用户 ID？

A: 
1. 在飞书中，点击用户头像
2. 查看「个人信息」
3. 用户 ID 显示在底部
