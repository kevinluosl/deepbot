# 飞书机器人配置指南

本文档介绍如何配置 DeepBot 的飞书连接器，使其能够通过飞书接收和发送消息。

## 前置条件

1. 拥有飞书企业管理员权限
2. 已安装 DeepBot Terminal

## 配置步骤

### 1. 创建飞书企业自建应用

1. 访问[飞书开放平台](https://open.feishu.cn/)
2. 登录后，点击「创建企业自建应用」
3. 填写应用信息：
   - 应用名称：DeepBot（或自定义名称）
   - 应用描述：AI 助手机器人
   - 应用图标：上传应用图标（可选）

### 2. 获取应用凭证

1. 在应用详情页，进入「凭证与基础信息」
2. 记录以下信息：
   - **App ID**（应用 ID）
   - **App Secret**（应用密钥）

### 3. 配置应用权限

在「权限管理」页面，添加以下权限：

**消息与群组**：
- `im:message` - 获取与发送单聊、群组消息
- `im:message:send_as_bot` - 以应用的身份发消息
- `im:chat` - 获取与管理群组信息（含读取）
- `im:resource` - 获取与上传图片或文件资源

**用户信息**：
- `contact:user.base:readonly` - 获取用户基本信息（真实姓名、头像等）
- `contact:user.basic_profile:readonly` - 获取用户基础档案
- `contact:contact.base:readonly` - 获取用户名字

**云文档（读写）**：
- `docx:document` - 创建和编辑云文档（含读取）
- `drive:drive` - 管理云空间文件（含读取，用于删除文档）
- `drive:file` - 访问云空间文件
- `sheets:spreadsheet:readonly` - 读取电子表格内容
- `docs:document.comment:create` - 在云文档中添加评论

也可以在「权限管理」页面点击「批量导入/导出权限」，直接粘贴以下 JSON 一键导入所有权限：

```json
{
  "scopes": {
    "tenant": [
      "contact:contact.base:readonly",
      "contact:user.base:readonly",
      "contact:user.basic_profile:readonly",
      "docs:document.comment:create",
      "docx:document",
      "drive:drive",
      "drive:file",
      "im:chat",
      "im:message",
      "im:message.group_at_msg:readonly",
      "im:message.group_msg",
      "im:message.p2p_msg:readonly",
      "im:message:send_as_bot",
      "im:resource",
      "sheets:spreadsheet:readonly"
    ],
    "user": []
  }
}
```

### 4. 在 DeepBot 中配置

**注意：配置事件订阅前，需要先在 DeepBot 中填入 App ID 和 App Secret，否则无法建立长连接。**

1. 启动 DeepBot Terminal
2. 进入「设置」→「连接器配置」
3. 选择「飞书」连接器
4. 填写配置信息：
   - **App ID**：从步骤 2 获取
   - **App Secret**：从步骤 2 获取
   - **机器人名称**：自定义显示名称（如 "DeepBot"）
5. 点击「保存」
6. 点击「启动」按钮启动连接器

### 5. 配置事件订阅

进入应用的「事件与回调」页面，需要分别配置两个子页面：

#### 事件配置

1. 点击「事件配置」标签
2. 订阅方式选择「使用长连接接收事件」
3. 点击「添加事件」，搜索并添加 `im.message.receive_v1`
4. 添加后，在事件列表中找到该事件，开通以下权限：
   - **接收群聊中@机器人消息事件** - 已开通
   - **读取用户发给机器人的单聊消息** - 已开通
   - **获取群组中所有消息（敏感权限）** - 已开通

#### 回调配置

1. 点击「回调配置」标签
2. 订阅方式同样选择「使用长连接接收事件」

**注意**：
- 长连接模式仅支持企业自建应用
- 无需配置请求地址或 Encrypt Key

### 7. 发布应用

## 使用说明

### 私聊使用

#### Pairing 模式（推荐）

1. 在飞书中搜索并添加机器人
2. 发送任意消息
3. 机器人会返回配对码（如：`ABC123`）
4. 管理员在 DeepBot Terminal 中批准：
   - 进入「设置」→「连接器配置」→「飞书」
   - 切换到「Pairing 管理」标签页
   - 找到对应的配对记录，点击「批准」按钮
5. 批准后用户即可正常对话

#### Allowlist 模式

1. 管理员在配置中添加用户 ID 到白名单
2. 白名单用户可直接与机器人对话

### 群组使用

1. 将机器人添加到群组
2. 在群组中 @机器人 发送消息
3. 机器人会回复消息

### 文件发送功能

DeepBot 支持通过飞书发送图片和文件：

1. **发送图片**：Agent 可以使用 `connector_send_image` 工具发送图片
   - 支持格式：JPG、PNG、GIF、BMP、WebP
   - 可选添加图片说明文字

2. **发送文件**：Agent 可以使用 `connector_send_file` 工具发送文件
   - 支持任意文件类型
   - 可自定义文件名

**使用示例**：
- "帮我把这个图片发送到飞书"
- "把这个文档发送给用户"
- "发送 /path/to/report.pdf 文件"

**注意**：文件发送功能仅在连接器会话中可用（即从飞书发起的对话）。

### 飞书云文档操作功能

DeepBot 支持通过飞书会话直接操作飞书云文档：

1. **创建文档**：`feishu_doc_create` - 在云空间创建新文档，自动将发送者添加为管理员
2. **读取文档**：`feishu_doc_get` - 获取文档基本信息和纯文本内容
3. **获取块列表**：`feishu_doc_get_blocks` - 获取文档所有块及内容，用于后续编辑
4. **追加内容**：`feishu_doc_append` - 在文档末尾追加新段落
5. **更新块**：`feishu_doc_update_block` - 更新指定块的文本内容
6. **删除块**：`feishu_doc_delete_blocks` - 删除文档中指定范围的块
7. **添加评论**：`feishu_doc_add_comment` - 在文档中添加全文评论
8. **删除文档**：`feishu_doc_delete_file` - 永久删除整篇文档（不可恢复）

**使用示例**：
- "帮我创建一篇标题为'项目周报'的飞书文档"
- "把这段内容追加到文档 doxcnXXX 末尾"
- "在文档里添加一条评论：已审阅"
- "删除文档第 2 到第 5 个块"

**权限要求**：需要在飞书开放平台配置云文档操作权限（见上方「云文档操作权限」部分）。

### 飞书文档和电子表格读取功能

DeepBot 支持自动读取飞书文档和电子表格内容：

1. **自动检测**：当用户在消息中发送飞书文档或电子表格链接时，DeepBot 会自动检测
2. **读取内容**：自动调用飞书 API 读取文档或表格的内容
3. **附加到消息**：将内容附加到用户消息中，供 AI 分析

**支持的文档类型**：
- 飞书文档（docx）
- 飞书旧版文档（docs）
- 飞书知识库（wiki）
- 飞书电子表格（sheets）- 会读取所有工作表的数据

**使用示例**：
- 用户：发送飞书文档链接 + "帮我总结这个文档"
- 用户：发送飞书电子表格链接 + "分析这个表格的数据"
- DeepBot：自动读取内容并进行分析

**权限要求**：需要在飞书开放平台配置相应权限（见上方「文档读取权限」部分）。
